Deploy on Self & Remote server

Running Multiple Odoo Instances in a single server slows down its performance. So, if you have multiple clients, then you can also deploy their instance in a remote server instead of the self server with the help of Odoo Saas Kit All in One Module.

The module facilitates you to choose between Self and Remote server as per the requirements of your business.

Running Odoo in Remote server means that you can segregate the servers for you(admin) and your clients for fast processing.

Add Subdomains(Domain Masking)

The module allows you to hide the actual domain name of a website from the URL field of a user's web browser in favour of another name. In simple words, multiple domains can be added for the client's instance.

Moreover, Admin has the flexibility to create a custom domain for the client as per the requirement.

Deploy Instance in any Odoo Version

The module Odoo Saas Kit All in One allows you to deploy Odoo Instance of your client in any Odoo Version. It means that no matter in which version your Odoo instance is running, you can deploy your Client's instance in any other Odoo Version.

Let's Understand this with an example:

Suppose Odoo Saas Kit is installed in your Odoo Version 13 and your clients want to run their Odoo in any other version, say V14, V15 or V12 then you can deploy their version in any asked Odoo Version using the custom plan addon of saas.

Customers can Manage the Backup

It offers the flexibility to the clients with the creation of a backup process for their instance by configuring the backup cycle. The client can configure a backup cycle to create a backup as hourly, daily, weekly, monthly.

As backup starts, the list of backups will be visible on the contract portal page. Clients can download the backuped file from the contract portal page.

    The instance of client can be managed on the same Odoo server on which Odoo SAAS Kit (Odoo server) is installed or the SAAS admin can choose to deploy the instance of client on a different remote server.
